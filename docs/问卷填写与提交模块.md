# 问卷填写与提交模块

## 模块概述

本模块面向问卷答题者（终端用户），提供公开问卷的访问、渲染、填写与提交全流程。支持通过 `shareToken` 访问已发布问卷，采集答案的同时记录设备信息与 IP 地域，并提供预览模式供编辑者试答。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 公开问卷访问 | 通过 `/s/{shareToken}` 访问已发布问卷 |
| 问卷数据获取 | 根据 token 返回问卷标题、描述、题目列表 |
| 答题渲染 | 动态加载各题型的答题组件 |
| 答案提交 | 校验必填、存储答案、关联版本 |
| 设备信息采集 | UA 解析出设备类型、OS、浏览器 |
| IP 地域解析 | 通过 IP 查询省份/城市 |
| 浏览量记录 | 同一 IP 24h 内去重计数 |
| 预览模式 | `?preview=1` 模式下不保存数据 |

---

## 关键文件说明

### 页面与 API

| 文件 | 职责 |
|------|------|
| `app/s/[token]/page.tsx` | 公开问卷页面：加载问卷、渲染题目、收集答案、提交 |
| `app/api/s/[token]/route.ts` | GET：根据 shareToken 获取问卷数据（预览模式需登录权限） |
| `app/api/s/[token]/submit/route.ts` | POST：提交答案，创建 Response 与 Answer 记录 |
| `app/api/s/[token]/view/route.ts` | POST：记录浏览量（IP 24h 去重） |

### 工具库

| 文件 | 职责 |
|------|------|
| `lib/ua-parser.ts` | 轻量级 User-Agent 解析器（无外部依赖） |
| `lib/geoip.ts` | IP 地址解析地域信息（基于 ip-api.com） |

### 答题组件

`components/questions/` 下各文件提供 `Response` 子组件，负责具体题型的答题交互：

| 题型 | 文件 | answer 值类型 |
|------|------|---------------|
| 单选题 | `single-choice.tsx` | `string`（选中选项的 label） |
| 多选题 | `multiple-choice.tsx` | `string[]`（选中选项的 label 数组） |
| 下拉选择 | `dropdown.tsx` | `string`（选中选项的 label） |
| 矩阵单选 | `matrix-single.tsx` | `Record<string, string>`（rowId → columnId） |
| 排序题 | `ranking.tsx` | `string[]`（选项 id 的顺序数组） |
| 评分题 | `rating.tsx` | `number`（1~N 的整数） |
| NPS | `nps.tsx` | `number`（0~10 的整数） |
| 单行文本 | `text.tsx` | `string` |
| 多行文本 | `textarea.tsx` | `string` |
| 性别 | `gender.tsx` | `string`（选项 id：`male`/`female`/`other`） |

## 核心逻辑/流程

### 1. 公开问卷访问流程

```
用户访问 /s/{token}
    ↓
GET /api/s/{token}
    ↓
查询 Survey：shareToken = token AND published = true
    ↓
返回 { title, description, questions[], settings }
    ↓
前端渲染答题页面
    ↓
（非预览模式）POST /api/s/{token}/view 记录浏览量
```

**预览模式**（`?preview=1`）的特殊处理：

```ts
// app/api/s/[token]/route.ts
if (isPreview) {
  // 需要登录，且是创建者或协作者
  const survey = await prisma.survey.findUnique({
    where: { id: token }, // 预览使用 survey id 作为 token
    include: { collaborators: { where: { userId } } },
  })
  const isOwner = survey.userId === userId
  const isCollaborator = survey.collaborators.length > 0
  if (!isOwner && !isCollaborator) return 403
  return survey
}

// 正常模式：只返回已发布问卷
const survey = await prisma.survey.findUnique({
  where: { shareToken: token, published: true },
})
```

### 3. 答案数据结构

各题型的 `answer.value` 存储格式：

```ts
// 单选题 / 下拉选择
"选项A"

// 多选题
["选项A", "选项B"]

// 矩阵单选：rowId -> columnId
{
  "row_xxx": "col_yyy",
  "row_zzz": "col_www"
}

// 排序题：按优先级排列的选项 id 数组
["opt_3", "opt_1", "opt_2"]

// 评分 / NPS / 数字

// 文本类
"用户输入的内容"

// 性别（特殊：存的是 option id，不是 label）
"male" | "female" | "other"
```

### 4. 提交 API 的 zod 校验 schema

```ts
const submitSchema = z.object({
  answers: z.record(
    z.string(),           // questionId
    z.union([
      z.string(),         // 单选、下拉、文本
      z.array(z.string()),// 多选、排序
      z.number(),         // 评分、NPS
      z.null(),           // 未答
      z.record(z.string(), z.string()), // MATRIX_SINGLE
    ])
  ),
  metadata: z.object({
    startedAt: z.string().datetime().nullable().optional(),
    deviceType: z.string().nullable().optional(),
    os: z.string().nullable().optional(),
    browser: z.string().nullable().optional(),
    source: z.string().nullable().optional(),
    referrer: z.string().nullable().optional(),
  }).optional(),
})
```

> **特别注意 `MATRIX_SINGLE`**：其 answer 类型是 `Record<string, string>`（`z.record(z.string(), z.string())`），即 `{ [rowId]: columnId }`。这是 schema 中唯一使用 `record` 类型的题型。

### 5. 设备信息采集（UA 解析）

`lib/ua-parser.ts` 是一个手写正则的轻量级解析器，不依赖 `ua-parser-js`：

```ts
export function parseUA(userAgent: string): ParsedUA {
  const ua = userAgent.toLowerCase()

  // 设备类型
  let deviceType: "desktop" | "mobile" | "tablet" = "desktop"
  if (/ipad|tablet|kindle|silk/.test(ua)) deviceType = "tablet"
  else if (/mobile|iphone|ipod|android.*mobile|windows phone/.test(ua))
    deviceType = "mobile"

  // 操作系统
  let os = "Other"
  if (/windows nt 10/.test(ua)) os = "Windows 10/11"
  else if (/macintosh|mac os x/.test(ua)) os = "macOS"
  else if (/iphone|ipad|ipod/.test(ua)) os = "iOS"
  else if (/android/.test(ua)) os = "Android"
  // ...

  // 浏览器
  let browser = "Other"
  if (/edg/.test(ua)) browser = "Edge"
  else if (/firefox/.test(ua)) browser = "Firefox"
  else if (/safari/.test(ua) && !/chrome/.test(ua)) browser = "Safari"
  else if (/chrome/.test(ua)) browser = "Chrome"

  return { deviceType, os, browser }
}
```

前端在提交时采集：

```ts
const ua = parseUA(navigator.userAgent)
const metadata = {
  startedAt: startedAt ? new Date(startedAt).toISOString() : null,
  deviceType: ua.deviceType,
  os: ua.os,
  browser: ua.browser,
  source: searchParams.get("source") || searchParams.get("utm_source") || null,
  referrer: document.referrer || null,
}
```

### 6. IP 地域解析

`lib/geoip.ts` 使用 ip-api.com 免费 API：

```ts
export async function lookupIP(ip: string): Promise<GeoIPResult> {
  // 内存缓存（24h）
  const cached = getCached(ip)
  if (cached) return cached

  // 跳过内网 IP
  if (ip === "::1" || ip.startsWith("127.") || ip.startsWith("192.168."))
    return { country: null, province: null, city: null }

  const res = await fetch(
    `http://ip-api.com/json/${ip}?lang=zh-CN&fields=status,message,country,regionName,city`,
    { signal: controller.signal } // 3s 超时
  )

  const result = {
    country: data.country || null,
    province: data.regionName || null,  // ip-api 的 regionName 对应省份
    city: data.city || null,
  }
  setCache(ip, result)
  return result
}
```

> IP 解析失败时**静默忽略**，不阻塞提交流程。

### 7. 浏览量记录

`POST /api/s/[token]/view`：

```ts
// 同一 IP 24 小时内只计一次
const oneDayAgo = new Date(Date.now() - 24 * 60 * 60 * 1000)
const existing = await prisma.surveyView.findFirst({
  where: { surveyId: survey.id, ip, viewedAt: { gte: oneDayAgo } },
})

if (!existing) {
  await prisma.surveyView.create({
    data: { surveyId: survey.id, ip, country, province, city },
  })
}

// 返回总浏览量
const totalViews = await prisma.surveyView.count({ where: { surveyId: survey.id } })
```

### 8. 预览模式（不保存数据）

```ts
const isPreview = searchParams.get("preview") === "1"

// 1. 加载问卷时区分 URL
const url = isPreview ? `/api/s/${token}?preview=1` : `/api/s/${token}`

// 2. 不记录浏览量
if (!isPreview) {
  fetch(`/api/s/${token}/view`, { method: "POST" }).catch(() => {})
}

// 3. 提交时前端模拟
if (isPreview) {
  await new Promise((resolve) => setTimeout(resolve, 500))
  setSubmitted(true)
  return
}
```

预览提交成功后显示提示：


## 重要实现细节

### 开始时间记录

使用 `sessionStorage` 防止刷新重置：

```ts
const stored = sessionStorage.getItem(`survey_start_${token}`)
const startTime = stored ? Number(stored) : Date.now()
if (!stored) {
  sessionStorage.setItem(`survey_start_${token}`, String(startTime))
}
```

### 答题组件的注册机制

所有题型通过 `lib/questions/registry.ts` 统一注册，页面通过 `getQuestionDef(question.type).Response` 动态获取答题组件：

```tsx
const def = getQuestionDef(question.type)
const ResponseComponent = def.Response

return (
  <ResponseComponent
    question={question}
    order={index + 1}
    showNumber={survey.settings?.showQuestionNumber ?? true}
    value={answers[q.id]}
    onChange={(v) => setAnswer(q.id, v)}
  />
)
```

### 矩阵单选的移动端适配

`matrix-single.tsx` 的 `Response` 组件提供了双布局：

- **桌面端**：传统表格布局，每行每列一个 radio
- **移动端**：卡片式纵向布局，每行一个卡片，内部列出所有列选项

```tsx
{/* 桌面端 */}
<div className="mt-4 hidden overflow-hidden rounded-lg border md:block">
  <table>...</table>
</div>

{/* 移动端 */}
<div className="mt-4 space-y-3 md:hidden">
  {rows.map((row) => (
    <div key={row.id} className="rounded-lg border bg-card p-4">
      {/* 纵向展示列选项 */}
    </div>
  ))}
</div>
```

### 性别题型的特殊存储

与其他选择题不同，性别题的 answer 存储的是 **option id**（`male`/`female`/`other`），而非 label（"男"/"女"/"其他"）。这在结果统计时需要特殊处理（详见结果统计模块文档）。
