# 问卷发布与版本管理

## 模块概述

本模块负责问卷的发布状态控制与版本快照管理。核心设计思路是：**问卷编辑态与发布态分离**——编辑者可以持续修改问卷内容，而每次"发布"会创建一个不可变的版本快照，对外分享的问卷始终基于某个特定版本，确保答题者看到的内容稳定一致。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 发布/取消发布 | 切换问卷的 `published` 布尔状态 |
| 版本创建 | 将当前问卷的题目数据快照保存为 `SurveyVersion` |
| 版本列表 | 查看历史版本及每版本的回答数 |
| 版本详情 | 查看某个版本的完整题目快照 |
| 版本回滚 | 将问卷的当前版本指向某个历史版本 |
| 版本删除 | 删除非当前版本的历史快照 |

---

## 关键文件说明

### 后端 API

| 文件 | 职责 |
|------|------|
| `app/api/surveys/[id]/publish/route.ts` | POST：切换 `published` 布尔值（仅创建者） |

## 核心逻辑/流程

### 1. 发布流程（生成 shareToken、创建版本快照）

问卷在创建时即通过 Prisma 的 `@default(cuid())` 自动生成唯一的 `shareToken`：

```prisma
model Survey {
  shareToken String @unique @default(cuid())
  published  Boolean @default(false)
  currentVersionId String?  // 当前发布的版本ID
  // ...
}
```

**发布新版本** 的完整流程（`POST /api/surveys/[id]/versions`）：

```ts
// 1. 权限校验：仅问卷创建者可发布
if (survey.userId !== session.user.id) return 403

// 2. 空问卷禁止发布
if (survey.questions.length === 0) return 400

// 3. 递增版本号
const latestVersion = await prisma.surveyVersion.findFirst({
  where: { surveyId: id },
  orderBy: { version: "desc" },
  select: { version: true },
})
const newVersionNumber = (latestVersion?.version || 0) + 1

// 4. 创建版本快照（questions 字段存储完整题目 JSON）
const version = await prisma.surveyVersion.create({
  data: {
    surveyId: id,
    version: newVersionNumber,
    title: survey.title,
    description: survey.description,
    questions: survey.questions.map((q) => ({
      id: q.id, type: q.type, title: q.title,
      description: q.description, required: q.required,
      order: q.order, config: q.config,
    })),
  },
})

// 5. 更新问卷：标记为已发布，并指向最新版本
await prisma.survey.update({
  where: { id },
  data: { published: true, currentVersionId: version.id },
})
```

### 2. 版本管理

#### 版本列表（GET）

```ts
const versions = await prisma.surveyVersion.findMany({
  where: { surveyId: id },
  orderBy: { version: "desc" },
  include: { _count: { select: { responses: true } } },
})

// 标记当前版本
const versionsWithCurrent = versions.map((v) => ({
  ...v,
  isCurrent: v.id === survey.currentVersionId,
}))
```

#### 版本详情（GET）

返回单个版本的完整数据，包括 `questions` JSON 快照和回答计数。

#### 版本回滚（POST）

```ts
// 仅更新 Survey.currentVersionId，不修改 SurveyVersion 表
await prisma.survey.update({
  where: { id },
  data: { currentVersionId: version.id },
})
```

> 回滚是**轻量级操作**：只切换指针，不复制数据。回滚后，新提交的答案将关联到被回滚的版本。

#### 版本删除（DELETE）

```ts
// 禁止删除当前版本
if (versionId === survey.currentVersionId) return 400

await prisma.surveyVersion.delete({ where: { id: versionId } })
```

## 数据结构

### SurveyVersion 模型

```prisma
model SurveyVersion {
  id          String   @id @default(cuid())
  surveyId    String
  version     Int      // 版本号 v1, v2, v3...
  title       String   // 发布时的问卷标题
  description String?  // 发布时的问卷描述
  questions   Json     // 完整的题目数据快照（含 config）
  publishedAt DateTime @default(now())
  createdAt   DateTime @default(now())

  survey    Survey     @relation(fields: [surveyId], references: [id], onDelete: Cascade)
  responses Response[] // 关联到该版本的回答

  @@unique([surveyId, version])
  @@index([surveyId])
  @@index([publishedAt])
}
```

### Response 与版本的关联

```prisma
model Response {
  id        String   @id @default(cuid())
  surveyId  String
  versionId String   // 必须关联到具体版本
  // ...
  version SurveyVersion @relation(fields: [versionId], references: [id], onDelete: Cascade)
}
```

每个 `Response` 必须关联一个 `SurveyVersion`，这是数据统计时版本隔离的基础。

## 重要实现细节

### 取消发布逻辑

`POST /api/surveys/[id]/publish` 的实现极其简单——只是翻转布尔值：

```ts
const updated = await prisma.survey.update({
  where: { id },
  data: { published: !survey.published },
})
```

- 取消发布时，`currentVersionId` **不会被清空**
- 再次发布时，会**创建新版本**，而不是恢复旧状态
- 取消发布后，外部用户通过 `shareToken` 访问会返回 404（因为查询条件包含 `published: true`）

### 发布状态与问卷状态的关系

| 状态组合 | 含义 |
|----------|------|
| `published = false`, `currentVersionId = null` | 未发布，无版本 |
| `published = true`, `currentVersionId = "xxx"` | 已发布，有当前版本（正常态） |
| `published = false`, `currentVersionId = "xxx"` | 已取消发布，但保留版本记录（可重新发布） |

### 权限模型

- **发布/删除/回滚**：仅问卷创建者（`userId === session.user.id`）
- **查看版本列表/详情**：创建者或协作者（`canViewResults`）

### 已知问题

**列表页发布不创建版本导致外键错误**

早期的问卷列表页有一个"快速发布"按钮，直接调用 `POST /api/surveys/[id]/publish` 切换 `published` 状态，但**没有创建 `SurveyVersion`**。这会导致：

1. 问卷标记为 `published = true`
2. 但 `currentVersionId` 仍为 `null`
3. 用户提交答案时，`versionId` 写入空字符串或 null
4. 触发数据库外键约束错误

**当前缓解措施**：隐藏列表页的发布按钮，强制用户进入编辑器通过"版本管理"弹窗进行规范发布（该流程会正确创建版本）。

**根本修复方向**：统一发布入口，或让 `publish` API 自动创建首个版本（如果尚无版本）。

## 前端交互：版本管理弹窗

`version-dialog.tsx` 提供了完整的版本管理 UI：

```tsx
<Dialog>
  {/* 顶部：发布按钮（仅所有者） */}
  {isOwner && (
    <Button onClick={handlePublish}>
      {published ? "发布新版本" : "首次发布"}
    </Button>
  )}

  {/* 中部：历史版本列表 */}
  {versions.map((v) => (
    <div key={v.id} className={v.isCurrent ? "border-primary" : ""}>
      <span>v{v.version}</span>
      {v.isCurrent && <span>当前</span>}
      <span>{v._count.responses} 份回答</span>
      {!v.isCurrent && (
        <>
          <Button onClick={() => handleRollback(v.id)}>回滚</Button>
          <Button onClick={() => handleDelete(v.id)}>删除</Button>
        </>
      )}
    </div>
  ))}
</Dialog>
```

弹窗打开时通过 `requestAnimationFrame` 延迟加载版本列表，避免 effect 同步 setState 的 React 警告。
