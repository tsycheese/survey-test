# 题型系统模块

## 模块概述

题型系统是问卷编辑器的核心扩展层，通过**注册中心模式（Registry Pattern）**将各种题型统一抽象为相同的接口。每种题型只需实现一套标准组件（编辑器配置、画布预览、答题组件），即可被编辑器自动识别和渲染，实现题型的即插即用。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 题型注册中心 | 集中管理所有题型定义，通过 `type` 字段动态查找 |
| 题型三件套 | 每种题型提供 Editor（配置）、Canvas（画布预览）、QuestionCard（编辑态卡片）、Response（答题态） |
| 选项管理 | 支持选项的增删改、列数设置、图片上传 |
| 默认题目生成 | 通过 `defaultQuestion(order)` 工厂函数生成带默认配置的初始题目 |
| 题型分类展示 | 按选择类、文本输入、高级题型、矩阵、个人信息、联系方式分组 |

| 文件路径 | 职责 |

## 题型注册机制

### QuestionDef 接口定义

```ts
type QuestionDef<Q extends Question = Question> = {
  type: Q["type"]                          // 题型标识
  category: QuestionCategory               // 所属分类
  label: string                            // 显示名称
  icon: React.ElementType                  // Lucide 图标组件
  defaultQuestion: (order: number) => Q    // 工厂函数

  // ===== 四件套组件 =====
  Canvas: React.FC<{ question: Q; selected: boolean; onUpdate?: (question: Q) => void }>
  Editor: React.FC<{ question: Q; onChange: (question: Q) => void; onSave?: (question: Q) => void }>
  QuestionCard: React.FC<{
    question: Q; selected: boolean; order: number;
    showNumber?: boolean; readOnly?: boolean;
    onUpdate?: (question: Q) => void;
    onTitleChange?: (title: string) => void;
    onTitleBlur?: (title: string) => void;
    onDescriptionChange?: (description: string) => void;
    onDescriptionBlur?: (description: string) => void;
    onOptionChange?: (question: Q) => void;
    onFocusQuestion?: () => void;
  }>
  Response: React.FC<{
    question: Q; order: number; showNumber?: boolean;
    value: unknown; onChange: (value: unknown) => void;
  }>
}
```

### 注册中心 (`lib/questions/registry.ts`)

```ts
import { singleChoiceDef } from "@/components/questions/single-choice"
import { multipleChoiceDef } from "@/components/questions/multiple-choice"
// ... 导入所有题型

export const QUESTION_REGISTRY: Record<QuestionType, QuestionDef> = {
  SINGLE_CHOICE: singleChoiceDef as QuestionDef,
  MULTIPLE_CHOICE: multipleChoiceDef as QuestionDef,
  TEXT: textDef as QuestionDef,
  // ... 共 18 种
} as const

export const QUESTION_DEFS = Object.values(QUESTION_REGISTRY).filter(Boolean)

// 根据 type 获取题型定义（找不到则抛错）
export function getQuestionDef(type: QuestionType): QuestionDef {
  const def = QUESTION_REGISTRY[type]
  if (!def) throw new Error(`Unknown question type: ${type}`)
  return def
}

// 创建默认题目
export function createQuestion(type: QuestionType, order: number): Question {
  return getQuestionDef(type).defaultQuestion(order) as Question
}
```

**使用方式：**
```ts
const def = getQuestionDef(question.type)
// 渲染画布预览
<def.Canvas question={question} selected={isSelected} onUpdate={...} />
// 渲染右侧编辑器
<def.Editor question={question} onChange={...} onSave={...} />
// 渲染答题组件
<def.Response question={question} value={value} onChange={...} />
```

## 每种题型的四件套组件

以 **单选题 (`SINGLE_CHOICE`)** 为例：

### 1. defaultQuestion — 默认题目工厂

```ts
defaultQuestion: (order) => ({
  id: nanoid(),
  type: "SINGLE_CHOICE",
  title: "新问题",
  required: false,
  order,
  config: {
    options: [
      { id: nanoid(), label: "选项 1" },
      { id: nanoid(), label: "选项 2" },
    ],
    columns: 1,
  },
})
```

### 2. Canvas — 画布预览（编辑态，内嵌在题目卡片中）

```tsx
Canvas: ({ question, onUpdate }) => {
  const { options, columns = 1 } = question.config
  const [editingOptId, setEditingOptId] = useState<string | null>(null)

  return (
    <div className="grid gap-2" style={{ gridTemplateColumns: `repeat(${columns}, minmax(0, 1fr))` }}>
      {options.map((opt) => (
        <div
          key={opt.id}
          className={cn(
            "flex items-center gap-3 rounded-lg border bg-card p-3",
            editingOptId === opt.id && "border-primary shadow-sm ring-1 ring-primary"
          )}
          onClick={() => setEditingOptId(opt.id)}
        >
          <div className="h-4 w-4 shrink-0 rounded-full border border-primary/50" />
          {editingOptId === opt.id ? (
            <input
              autoFocus
              value={opt.label}
              onChange={(e) => handleOptUpdate(opt.id, e.target.value)}
              onBlur={() => setEditingOptId(null)}
            />
          ) : (
            <span className="truncate font-medium">{opt.label}</span>
          )}
        </div>
      ))}
    </div>
  )
}
```

**特点：**
- 点击选项即可**就地编辑**标签文字。
- 使用 `pointer-events-none` 的固定边框容器避免编辑时边框变化导致的布局抖动。
- 编辑完成后 `onBlur` 自动保存。

### 3. Editor — 右侧属性面板配置

```tsx
Editor: ({ question, onChange, onSave }) => {
  const { options, columns = 1 } = question.config

  function updateOption(id: string, label: string) {
    onChange({
      ...question,
      config: { ...question.config, options: options.map((o) => o.id === id ? { ...o, label } : o) }
    })
  }

  function addOption() {
    onSave?.({
      ...question,
      config: {
        ...question.config,
        options: [...options, { id: nanoid(), label: `选项${options.length + 1}` }],
      },
    })
  }

  function removeOption(id: string) {
    if (options.length <= 1) return   // 至少保留 1 个选项
    onChange({ ...question, config: { ...question.config, options: options.filter((o) => o.id !== id) } })
  }

  return (
    <div className="space-y-4">
      {/* 列数选择 */}
      <Select value={columns.toString()} onValueChange={(v) => { ... }}>
        {[1, 2, 3, 4].map((n) => <SelectItem key={n} value={n.toString()}>{n} 列</SelectItem>)}
      </Select>

      {/* 选项列表：可编辑、可删除 */}
      {options.map((opt) => (
        <div key={opt.id} className="flex items-center gap-2">
          <div className="h-3.5 w-3.5 rounded-full border border-border" />
          <Input value={opt.label} onChange={(e) => updateOption(opt.id, e.target.value)} />
          <Button variant="ghost" size="icon" onClick={() => removeOption(opt.id)}>
            <Trash2 className="h-3.5 w-3.5" />
          </Button>
        </div>
      ))}

      {/* 添加选项 */}
      <Button variant="outline" size="sm" onClick={addOption} className="w-full">
        <PlusCircle className="mr-1.5 h-3.5 w-3.5" /> 添加选项
      </Button>
    </div>
  )
}
```

### 4. QuestionCard — 编辑态完整卡片

与 `Canvas` 类似，但额外集成：
- `QuestionTitle` 可编辑组件（支持标题/描述的就地编辑）
- `onTitleChange` / `onTitleBlur` 等回调
- `onFocusQuestion` 回调（点击题目时触发选中+锁定）

### 5. Response — 答题态组件

```tsx
Response: ({ question, order, showNumber = true, value, onChange }) => {
  const { options, columns = 1 } = question.config
  const selectedValue = value as string | undefined

  return (
    <div className="relative px-3 py-3">
      <QuestionTitleReadonly order={order} showNumber={showNumber} title={question.title} ... />
      <div className="grid gap-2" style={{ gridTemplateColumns: `repeat(${columns}, minmax(0, 1fr))` }}>
        {options.map((opt) => {
          const isSelected = selectedValue === opt.label
          return (
            <button
              key={opt.id}
              type="button"
              onClick={() => onChange(opt.label)}
              className={cn(
                "flex items-center gap-3 rounded-lg border p-3",
                isSelected ? "border-primary bg-primary/5" : "border-border bg-card"
              )}
            >
              <div className={cn("h-4 w-4 rounded-full border", isSelected ? "bg-primary" : "border-primary/50")}>
                {isSelected && <div className="h-1.5 w-1.5 rounded-full bg-white" />}
              </div>
              <span className="truncate font-medium">{opt.label}</span>
            </button>
          )
        })}
      </div>
    </div>
  )
}
```

**注意：** `Response` 组件中，单选题的 `value` 存储的是**选项的 `label` 字符串**，而非 `id`。

## 选项管理（增删改、排序）

### 通用模式

所有带选项的题型（单选、多选、下拉、排序、矩阵、图片选择）遵循相同的选项管理范式：

1. **选项 ID 生成**：使用 `nanoid()` 生成唯一 ID。
2. **就地编辑**：点击选项标签即可进入编辑模式，`onBlur` 或 `Enter` 保存。
3. **增删限制**：
   - 单选/多选/下拉：至少保留 **1** 个选项。
   - 图片选择：至少保留 **2** 个选项。
   - 矩阵列：至少保留 **2** 个列。
4. **列数设置**：选择类题型支持 `columns` 配置（1-4 列），通过 CSS Grid 实现响应式布局。

### 选项编辑的防抖优化

```ts
const originalLabelsRef = useRef<Map<string, string>>(new Map())

// onFocus：记录原始值
onFocus={() => originalLabelsRef.current.set(opt.id, opt.label)}

// onBlur：对比原始值，有变化才触发 onSave
onBlur={(e) => {
  const original = originalLabelsRef.current.get(opt.id)
  if (e.target.value !== original) {
    onSave?.(updatedQuestion)
  }
}}
```

## 特殊题型的实现细节

### 1. GENDER（性别选择）— 存储 option ID 而非 label

**关键差异：** 性别题的 `Response` 组件中，`value` 存储的是**选项的 `id`**（如 `"male"`、`"female"`、`"other"`），而不是 `label`。

```tsx
// components/questions/gender.tsx
Response: ({ question, order, value, onChange }) => {
  const defaultOptions = [
    { id: "male", label: "男" },
    { id: "female", label: "女" },
    { id: "other", label: "其他" },
  ]

  return (
    <div className="mt-3 space-y-2">
      {defaultOptions.map((opt) => (
        <label
          key={opt.id}
          className={cn(
            "flex cursor-pointer items-center gap-3 rounded-lg border bg-card p-3",
            value === opt.id ? "border-primary bg-primary/5" : "hover:bg-muted/50"
          )}
        >
          <input
            type="radio"
            name={`gender-${question.id}`}
            value={opt.id}
            checked={value === opt.id}      // 比较的是 id！
            onChange={() => onChange?.(opt.id)}  // 传递的也是 id！
            className="sr-only"
          />
          {/* ... */}
          <span className="font-medium">{opt.label}</span>
        </label>
      ))}
    </div>
  )
}
```

**设计原因：**
- 性别选项的标签可能因语言/业务变化（如 "男/女/其他" 变为 "Male/Female/Other"），但 ID 保持稳定。
- 数据统计时以 ID 为维度，不受文案变更影响。
- 性别题的选项**固定不可编辑**，Editor 中仅做展示说明。

### 2. TEXT（单行文本）— 格式校验

```ts
const FORMAT_OPTIONS = [
  { value: "any", label: "不限", placeholder: "请输入文本" },
  { value: "number", label: "数字", placeholder: "请输入数字" },
  { value: "date", label: "日期", placeholder: "YYYY-MM-DD" },
  { value: "email", label: "电子邮件", placeholder: "example@email.com" },
  { value: "phone", label: "手机号", placeholder: "13800138000" },
]

function validateInput(value: string, format: TextFormat): boolean {
  if (format === "any") return true
  if (format === "number") return /^\d+$/.test(value)
  if (format === "date") return /^\d{4}-\d{2}-\d{2}$/.test(value)
  if (format === "email") return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)
  if (format === "phone") return /^1\d{10}$/.test(value)
  return true
}
```

- Canvas 中展示 disabled 的 Input，并实时校验格式。
- Response 中答题时同样进行前端格式校验，错误时显示红色边框与提示文字。

### 3. RATING（评分题）— 星标交互

```tsx
Response: ({ question, value, onChange }) => {
  const [hoverValue, setHoverValue] = useState<number | null>(null)

  return (
    <div className="flex items-center gap-2">
      {Array.from({ length: max ?? 5 }, (_, i) => {
        const starValue = i + 1
        const isFilled = (hoverValue !== null ? hoverValue : (value as number) || 0) >= starValue
        return (
          <button
            key={i}
            onClick={() => onChange?.(starValue)}
            onMouseEnter={() => setHoverValue(starValue)}
            onMouseLeave={() => setHoverValue(null)}
          >
            <Star className={cn("h-6 w-6", isFilled ? "fill-yellow-400 text-yellow-400" : "text-muted-foreground")} />
          </button>
        )
      })}
    </div>
  )
}
```

- 支持 `hoverValue` 实现悬停预览效果。
- 可配置最大分值（3/5/7/10）与两端标签。

### 4. MATRIX_SINGLE（矩阵单选）— 响应式双布局

```tsx
Response: ({ question, value, onChange }) => {
  const selectedValues = (value as Record<string, string>) || {}
  const handleSelect = (rowId: string, columnId: string) => {
    onChange?.({ ...selectedValues, [rowId]: columnId })
  }

  return (
    <>
      {/* 桌面端：表格布局 */}
      <div className="hidden overflow-hidden rounded-lg border md:block">
        <table className="w-full">{/* thead + tbody */}</table>
      </div>

      {/* 移动端：卡片式纵向布局 */}
      <div className="mt-4 space-y-3 md:hidden">
        {rows.map((row) => (
          <div key={row.id} className="rounded-lg border bg-card p-4">
            <div className="mb-3 text-sm font-medium">{row.label}</div>
            <div className="space-y-2">
              {columns.map((col) => (
                <label key={col.id} className="...">
                  <input
                    type="radio"
                    name={`matrix-${question.id}-${row.id}`}
                    checked={selectedValues[row.id] === col.id}
                    onChange={() => handleSelect(row.id, col.id)}
                  />
                  <span>{col.label}</span>
                </label>
              ))}
            </div>
          </div>
        ))}
      </div>
    </>
  )
}
```

- 桌面端使用 `<table>` 展示矩阵，移动端使用卡片式纵向列表。
- `value` 为 `Record<rowId, columnId>` 对象，每行只能选一个列。

### 5. IMAGE_SINGLE_CHOICE（图片单选）— Cloudinary 图片上传

```ts
type ImageOption = {
  id: string
  imageUrl: string
  publicId?: string      // Cloudinary 资源 ID
  title?: string
  label?: string
  width?: number
  height?: number
}
```

**图片上传流程：**
1. 点击选项卡片的图片区域，触发隐藏的 `<input type="file">`。
2. 选择图片后调用 `uploadToCloudinary(file)` 上传。
3. 上传成功后更新选项的 `imageUrl`、`publicId`、`width`、`height`。
4. 删除图片时，若存在 `publicId`，先调用 `deleteFromCloudinary(publicId)` 清理云端资源。

**布局配置：**
- 支持 5 种图片比例：`3:4`（默认）、`4:3`、`1:1`、`16:9`、`9:16`。
- 可独立控制是否显示 `title` 和 `label`。
- 选项卡片使用 CSS `aspect-ratio` 保持图片区域比例一致。

## 题目标题组件

### QuestionTitle（编辑态）

```tsx
export function QuestionTitle({ order, showNumber, title, description, required, onChange, onBlur, ... }) {
  const [editingTitle, setEditingTitle] = useState(false)
  const [editingDesc, setEditingDesc] = useState(false)

  return (
    <div className="relative flex flex-col gap-1">
      {/* 标题：点击进入编辑，textarea 支持多行 */}
      {editingTitle ? (
        <textarea autoFocus value={title} onChange={...} onBlur={...} />
      ) : (
        <div onClick={() => setEditingTitle(true)} className="cursor-text hover:border-dashed">
          {showNumber && <span>{order}.</span>}
          {title || "请输入题目标题"}
          {required && <span className="text-red-500">*</span>}
        </div>
      )}

      {/* 描述：同样支持就地编辑 */}
      {editingDesc ? (...) : (...)}
    </div>
  )
}
```

**设计要点：**
- 使用 `<textarea>` 而非 `<input>`，支持题目标题的多行输入。
- `onFocus` 时通过 `setSelectionRange(length, length)` 将光标移至末尾。
- `onBlur` 时对比原始值，有变化才触发 `onBlur` 回调（避免无意义的 API 调用）。

### QuestionTitleReadonly（答题态）

纯展示组件，无编辑能力，用于 `Response` 组件中。

## 扩展新题型的步骤

如需新增一种题型，只需完成以下步骤：

1. **定义类型**（`lib/questions/types.ts`）：
   - 添加 `XxxConfig` 类型。
   - 添加 `XxxQuestion` 联合类型成员。
   - 更新 `Question` 联合类型。

2. **创建组件文件**（`components/questions/xxx.tsx`）：
   - 实现 `defaultQuestion` 工厂函数。
   - 实现 `Canvas`、`Editor`、`QuestionCard`、`Response` 四个组件。
   - 导出 `xxxDef: QuestionDef<XxxQuestion>`。

3. **注册到注册中心**（`lib/questions/registry.ts`）：
   - `import { xxxDef } from "@/components/questions/xxx"`
   - 在 `QUESTION_REGISTRY` 中添加映射：`XXX_TYPE: xxxDef as QuestionDef`

4. **API 校验**（`app/api/surveys/[id]/questions/route.ts` 等）：
   - 在 Zod Schema 的 `type` enum 中新增题型标识。

完成以上步骤后，新题型将自动出现在左侧题型面板、支持拖拽添加、画布渲染、属性配置和答题收集。
