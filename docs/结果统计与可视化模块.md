# 结果统计与可视化模块

## 模块概述

本模块为问卷创建者提供多维度的数据统计与可视化分析能力。基于 `Response` 和 `Answer` 数据，生成 KPI 概览、逐题统计图表、交叉分析透视表、原始数据详情表格，并支持 AI 智能总结。所有统计均支持按版本筛选，确保版本隔离。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 数据概览 | KPI 卡片、回收趋势、设备/OS 分布、地域分布 |
| 数据详情 | 原始回答表格、搜索、筛选、分页、批量删除、CSV 导出 |
| 统计图表 | 逐题分析、多种图表类型（饼图/柱状图/折线图） |
| 交叉分析 | 两题交叉透视表 |
| AI 总结 | 基于回答数据的 AI 洞察报告（独立路由） |
| 版本筛选 | 所有页面支持按版本筛选数据 |

### 2. 数据获取 Hook（use-results-data）

```ts
export function useResultsData(surveyId: string, versionId?: string | null) {
  const [data, setData] = useState<ResultsData | null>(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState<string | null>(null)

  // 自动根据 versionId 拼接 URL
  const url = versionId
    ? `/api/surveys/${surveyId}/responses?versionId=${versionId}`
    : `/api/surveys/${surveyId}/responses`

  // useEffect 自动加载
  // fetchData 作为 refetch 暴露
  return { data, loading, error, refetch: fetchData }
}
```

所有结果子页面都遵循相同的模式：
1. 调用 `useResultsData(id, selectedVersionId)`
2. 首次加载时默认选中最新版本
3. 版本切换时自动重新拉取数据

### 4. 数据详情（Details）

`details-tab.tsx` 是一个功能完整的数据表格：

#### 搜索

全局搜索匹配：设备类型、OS、浏览器、来源、省份、城市、任意答案内容。

```ts
if (search.trim()) {

    return r.answers.some((a) => String(a.value).toLowerCase().includes(s))

```

#### 筛选

支持 6 个维度 + 日期范围：

```ts
type Filters = {
  deviceType: string  // all | desktop | mobile | tablet
  os: string
  browser: string
  source: string
  province: string
  city: string

  endDate: Date | undefined

```

城市下拉会根据已选省份动态过滤（只显示该省份下的城市）。

#### 条件查询

支持基于题目答案的高级筛选：

```ts

  questionId: string
  type: "answered" | "unanswered" | "selected" | "not_selected"
  optionLabels?: string[]  // 用于 selected/not_selected
}
```

多选题的 `selected` 条件使用 **OR** 逻辑：包含任一已选选项即匹配。

#### 分页

前端分页，每页 20 条：

```ts

```

#### CSV 导出

```ts
function downloadCSV(data: ResultsData) {
  const headers = ["提交时间", "完成时长", "设备", "系统", "浏览器", "来源", "省份", "城市", ...questions]
  const rows = data.responses.map((r) => [...])
  const csv = [headers, ...rows].map((row) => row.map((cell) => `"${String(cell).replace(/"/g, '""')}"`).join(",")).join("\n")
  const blob = new Blob(["\uFEFF" + csv], { type: "text/csv;charset=utf-8;" })
  // 触发下载
}
```

> 添加 BOM（`\uFEFF`）确保 Excel 正确识别 UTF-8 编码。

#### 批量删除

选中行后批量调用 `DELETE /api/responses/{id}`，删除成功后调用 `onRefresh()` 刷新数据。

### 5. 统计图表（Charts）

`charts-tab.tsx` 逐题渲染统计卡片，根据题型选择不同的展示方式：

```tsx
function QuestionContent({ question, total }) {
  // 文本类：展示文本列表
  if (type === "TEXT" || type === "TEXTAREA" || type === "PHONE" || type === "EMAIL" || type === "NAME")
    return <TextAnswerList />

  // 评分类：展示平均分 + 分布条形图
  if (type === "RATING" || type === "NPS" || type === "CES" || type === "NUMBER")
    return <ScoreDistribution />

  // 矩阵单选：表格展示
  if (type === "MATRIX_SINGLE") return <MatrixTable />

  // 性别 / 单选 / 多选 / 下拉 / 排序 / 图片选择
  return <ChoiceStats />
}
```

#### 选择题统计（ChoiceStats）

```ts
const counts: Record<string, number> = {}
answers.forEach((a) => {
  const val = a.value
  if (Array.isArray(val)) {
    val.forEach((v) => { counts[String(v)] = (counts[String(v)] || 0) + 1 })
  } else {
    counts[String(val)] = (counts[String(val)] || 0) + 1
  }
})
```

#### 矩阵单选统计（MatrixTable）

```ts
const matrixCounts: Record<string, Record<string, number>> = {}
answers.forEach((a) => {
  const val = a.value as Record<string, string>
  Object.entries(val).forEach(([rowId, colId]) => {
    const rowLabel = rows.find((r) => r.id === rowId)?.label
    const colLabel = cols.find((c) => c.id === colId)?.label
    if (rowLabel && colLabel) {
      matrixCounts[rowLabel][colLabel] = (matrixCounts[rowLabel][colLabel] || 0) + 1
    }
  })
})
```

#### 图表组件（question-chart.tsx）

基于 echarts 封装，支持三种图表类型：

- **饼图**：环形饼图，带图例
- **柱状图**：带百分比 tooltip
- **折线图**：平滑曲线，带面积填充

支持暗黑模式自适应（监听 `document.documentElement.classList` 变化）。

### 6. 交叉分析（Cross）

`cross-tab.tsx` 实现两题交叉透视表：

```tsx
const [rowQId, setRowQId] = useState("")
const [colQId, setColQId] = useState("")
```

矩阵计算逻辑：

```ts
// 获取每道回答在该题上的值
const rowValues = getQuestionValues(rowQ, data.responses) // Map<responseId, string[]>
const colValues = getQuestionValues(colQ, data.responses)

// 初始化计数矩阵
const counts: Record<string, Record<string, number>> = {}

// 填充（多选题值展开为笛卡尔积）
data.responses.forEach((r) => {
  const rVals = rowValues.get(r.id) || ["未回答"]
  const cVals = colValues.get(r.id) || ["未回答"]
  rVals.forEach((rv) => {
    cVals.forEach((cv) => {
      counts[rv][cv] = (counts[rv][cv] || 0) + 1
    })
  })
})
```

表格展示：行标题 × 列标题 的计数矩阵，附带百分比和行列合计。

### 7. AI 总结页面

`summary/page.tsx` 是独立路由，**未在 sidebar 中显示**。功能：

1. 首次进入且无缓存时，自动调用 `/api/ai/summarize` 生成报告
2. 使用 SSE（Server-Sent Events）流式接收 AI 输出
3. 本地缓存（`localStorage`）按 `surveyId + totalResponses` 维度缓存
4. 支持复制、重新生成

```ts
interface CachedSummary {
  content: string
  timestamp: number
  surveyId: string
  totalResponses: number  // 缓存失效条件：回答数变化时重新生成
}
```

### 8. GENDER 题型的特殊计数逻辑

**关键差异**：性别题的 answer value 存储的是 **option id**（`male`/`female`/`other`），而其他选择题存储的是 **option label**。

这在 `charts-tab.tsx` 的 `ChoiceStats` 中做了特殊处理：

```ts
// GENDER 类型的 answer value 存的是 option id，其他类型存的是 label
const matchKey = type === "GENDER" ? "id" : "label"

const chartData = displayOptions.map((opt) => ({
  name: opt.label || "(未命名选项)",
  count: counts[opt[matchKey]] || 0,  // GENDER 用 opt.id 匹配，其他用 opt.label
}))
```

如果不做这个区分，性别统计将永远显示为 0（因为 `counts` 的 key 是 `"male"`，而 `opt.label` 是 `"男"`）。

## 统计数据 API 详解

`GET /api/surveys/[id]/responses?versionId=xxx`

### 输入参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `versionId` | query | 可选，筛选特定版本的回答 |

### 输出结构

```ts
{
  survey: { title, description },
  totalResponses: number,
  totalViews: number,
  completionRate: number,      // 保留 1 位小数
  avgCompletionTime: number,   // 秒，取整
  deviceStats: [{ name, count }],
  osStats: [{ name, count }],
  browserStats: [{ name, count }],
  sourceStats: [{ name, count }],
  locationStats: [{ name, count }],  // 按 count 降序
  countryStats: [{ name, count }],
  dailyTrend: [{ date, views, responses }],
  versions: [{ id, version }],
  currentVersionId: string | null,
  questions: [{ id, title, type, config, answers: [{ value }] }],
  responses: [{ id, createdAt, startedAt, completedAt, deviceType, os, browser, source, referrer, ip, country, province, city, answers: [{ questionId, value }] }]
}
```

### 版本题目数据回退

如果指定了 `versionId`，API 优先使用版本快照中的题目数据：

```ts
if (versionId) {
  const version = await prisma.surveyVersion.findUnique({ where: { id: versionId } })
  if (version && version.surveyId === id) {
    questions = version.questions as { id, title, type, config }[]
  }
}
// 兜底：使用当前 Question 表数据
if (questions.length === 0) {
  questions = await prisma.question.findMany({ where: { surveyId: id } })
}
```

这确保了回滚到旧版本后，统计图表展示的是与该版本回答对应的题目结构。

## 重要实现细节

### echarts 主题自适应

所有 echarts 组件都监听 `document.documentElement` 的 `class` 属性变化（暗黑模式切换）：

```ts
const observer = new MutationObserver(() => {
  instance.setOption(buildOption(), true)
})
observer.observe(document.documentElement, {
  attributes: true,
  attributeFilter: ["class"],
})
```

### 地图数据加载

`location-stats.tsx` 从阿里云 DataV CDN 加载中国 GeoJSON，有内存缓存：

```ts
const MAP_URLS = {
  china: "https://geo.datav.aliyun.com/areas_v3/bound/100000_full.json",
}
```

### 性能考虑

- 统计数据 API 一次性返回所有回答（不做分页），由前端处理分页和筛选
- 答案数据通过 `questions[].answers` 扁平化预聚合，减少前端计算
- echarts 实例在组件卸载时正确 `dispose()`，避免内存泄漏
