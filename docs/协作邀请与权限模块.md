# 协作邀请与权限模块

## 1. 模块概述

「协作邀请与权限模块」为问卷系统提供多人协作编辑与权限管理能力。问卷创建者（Owner）可以生成邀请码或邀请链接，邀请其他用户以不同权限加入问卷协作。模块涵盖：

- **邀请码管理**：创建、删除、分享邀请码，支持有效期和使用次数限制。
- **邀请链接直达**：通过 `/invite/{surveyId}/{code}` 路径一键加入。
- **协作者权限控制**：仅查看 / 可编辑 / 可查看结果三级权限模型。
- **权限检查工具**：服务端统一封装 `checkSurveyPermission`，用于 API 路由鉴权。
- **「我参与的问卷」**：协作者可在独立页面查看所有参与协作的问卷。
- **实时协作辅助**：加入/离开协作、题目锁定/解锁等 Presence Channel 配套接口。

整体架构遵循「创建者全权管理 + 协作者分级授权」的原则，所有敏感操作均校验当前用户身份与权限。

## 4. 邀请码机制

### 4.1 生成规则

- **长度**：6 位大写字母数字混合码。
- **生成方式**：使用 `nanoid(6)` 生成，并做易混淆字符替换：
  - `0` / `O` → `1`
  - `I` / `L` → `J`
- **唯一约束**：数据库 `SurveyInvite.code` 字段设有 `@unique` 索引。

### 4.2 有效期配置

创建时可选择：

| 选项 | 说明 |
|------|------|
| 1天 | `expiresAt = now + 1 day` |
| 7天 | `expiresAt = now + 7 days` |
| 30天 | `expiresAt = now + 30 days` |
| 永久 | `expiresAt = null` |

验证时检查：`if (invite.expiresAt && new Date() > invite.expiresAt)` 则返回过期错误。

### 4.3 使用次数限制

- `maxUses`：可选字段，当前默认不设置（即无限次）。
- `usedCount`：每次成功加入后 `increment: 1`。
- 验证逻辑：`if (invite.maxUses && invite.usedCount >= invite.maxUses)` 则返回已达上限错误。

### 4.4 权限配置

创建邀请码时绑定 `permissions` JSON 对象：

```json
{
  "canEdit": false,
  "canViewResults": false
}
```

- **仅查看**：`canEdit = false`，协作者只能预览问卷，不能修改。
- **可编辑**：`canEdit = true`，同时 `canViewResults = true`，协作者可编辑问卷并查看统计结果。

> 前端 UI 将权限简化为「仅查看」与「可编辑」两档；Owner 创建邀请码时只能授予不高于自身拥有的权限（如自身无编辑权则不能授予编辑权）。

## 5. 邀请链接流程

```
Owner 创建邀请码
    │
    ▼
生成 6 位 code → 存入 SurveyInvite（含 surveyId、permissions、expiresAt）
    │
    ▼
复制链接：/invite/{surveyId}/{code}
    │
    ▼
分享链接给被邀请人
    │
    ▼
被邀请人点击链接 ──────────────────────► /invite/{surveyId}/{code}
    │                                        │
    │                                        ▼
    │                              页面加载时自动 POST /api/invites/validate
    │                              验证：存在性 / 过期 / 次数 / 是否已加入 / 是否 Owner
    │                                        │
    │                              ┌─────────┴─────────┐
    │                              ▼                   ▼
    │                          验证通过              验证失败
    │                              │                   │
    │                              ▼                   ▼
    │                        展示问卷标题与权限      显示错误信息
    │                        用户点击「确认加入」    提供返回按钮
    │                              │
    │                              ▼
    │                        POST /api/invites/join
    │                        创建 SurveyCollaborator 记录
    │                        usedCount + 1
    │                              │
    │                              ▼
    │                        跳转 /surveys/{surveyId}/edit
    │
    ▼
被邀请人也可通过 /invite 页面手动输入 surveyId + code 加入
```

## 6. 协作者权限模型

### 6.1 权限层级

系统采用三级权限模型，以布尔字段组合实现：

| 权限级别 | `canEdit` | `canViewResults` | 能力说明 |
|----------|-----------|------------------|----------|
| 仅查看 | `false` | `false` | 可预览问卷，不可编辑，不可查看统计结果 |
| 可编辑 | `true` | `true` | 可编辑问卷内容，可查看统计结果 |
| 可查看结果 | — | `true` | 目前与「可编辑」绑定，编辑权限自动包含查看结果权限 |

### 6.2 权限继承规则

- **Owner**：拥有全部权限（`isOwner = true`），不受 `SurveyCollaborator` 记录限制。
- **协作者**：权限以 `SurveyCollaborator` 表中对应记录为准。
- **访客**：未登录或无协作关系的用户，无任何权限。

### 6.3 权限变更约束

- 只有 **Owner** 可以修改协作者权限或移除协作者。
- 协作者自身不能提升自己的权限。
- 创建邀请码时，当前用户只能授予 **不高于自身** 的权限（前端在 `collaboration-dialog.tsx` 中做此校验）。

## 7. 权限检查工具（`lib/collaboration.ts`）

### 7.1 `checkSurveyPermission`

统一封装的服务端权限查询函数，所有需要鉴权的 API 均可调用。

```typescript
import { prisma } from "@/prisma"

export interface CollaborationInfo {
  isOwner: boolean
  isCollaborator: boolean
  canEdit: boolean
  canViewResults: boolean
}

export async function checkSurveyPermission(
  surveyId: string,
  userId: string
): Promise<CollaborationInfo> {
  const survey = await prisma.survey.findUnique({
    where: { id: surveyId },
    include: {
      collaborators: {
        where: { userId }, // 只查询当前用户的协作记录
      },
    },
  })

  if (!survey) {
    return {
      isOwner: false,
      isCollaborator: false,
      canEdit: false,
      canViewResults: false,
    }
  }

  const isOwner = survey.userId === userId
  const collaborator = survey.collaborators[0]

  return {
    isOwner,
    isCollaborator: !!collaborator,
    canEdit: isOwner || collaborator?.canEdit || false,
    canViewResults: isOwner || collaborator?.canViewResults || false,
  }
}
```

**使用场景示例**：

```typescript
// 在 API Route 中快速鉴权
const perm = await checkSurveyPermission(surveyId, session.user.id)
if (!perm.canEdit) {
  return NextResponse.json({ error: "Forbidden" }, { status: 403 })
}
```

### 7.2 `logSurveyAction`

记录问卷操作日志，便于审计与追溯。

```typescript
export async function logSurveyAction(
  surveyId: string,
  userId: string,
  action: string,
  details?: Record<string, unknown>
) {
  await prisma.surveyLog.create({
    data: {
      surveyId,
      userId,
      action,
      details: (details || {}) as never,
    },
  })
}
```

## 9. 数据模型

### 9.1 `SurveyCollaborator` — 协作者关系

```prisma
model SurveyCollaborator {
  id             String   @id @default(cuid())
  surveyId       String
  userId         String
  canEdit        Boolean  @default(false)
  canViewResults Boolean  @default(false)
  invitedBy      String   // 邀请人（邀请码创建者）userId
  createdAt      DateTime @default(now())

  survey Survey @relation(fields: [surveyId], references: [id], onDelete: Cascade)
  user   User   @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([surveyId, userId]) // 同一用户同一问卷只能有一条记录
  @@index([surveyId])
  @@index([userId])
}
```

**关联**：
- `User.collaborations` ←→ `SurveyCollaborator.user`
- `Survey.collaborators` ←→ `SurveyCollaborator.survey`

### 9.2 `SurveyInvite` — 邀请码

```prisma
model SurveyInvite {
  id          String    @id @default(cuid())
  surveyId    String
  code        String    @unique // 6位邀请码，大写
  maxUses     Int?      // 最大使用次数，null 表示无限
  usedCount   Int       @default(0)
  expiresAt   DateTime? // 过期时间，null 表示永久
  permissions Json?     // { canEdit: boolean, canViewResults: boolean }
  createdBy   String    // 创建者 userId
  createdAt   DateTime  @default(now())

  survey Survey @relation(fields: [surveyId], references: [id], onDelete: Cascade)

  @@index([surveyId])
  @@index([code])
}
```

**关联**：
- `Survey.invites` ←→ `SurveyInvite.survey`

### 9.3 `Survey` 相关字段

```prisma
model Survey {
  id               String   @id @default(cuid())
  title            String
  // ...
  maxCollaborators Int      @default(10) // 协作者人数上限

  collaborators SurveyCollaborator[]
  invites       SurveyInvite[]
}
```

### 9.4 数据库索引汇总

| 表 | 索引/约束 | 用途 |
|----|-----------|------|
| `SurveyCollaborator` | `@@unique([surveyId, userId])` | 防止重复协作关系 |
| `SurveyCollaborator` | `@@index([surveyId])` | 快速查询某问卷的所有协作者 |
| `SurveyCollaborator` | `@@index([userId])` | 快速查询某用户参与的所有问卷 |
| `SurveyInvite` | `@@unique([code])` | 邀请码唯一性 |
| `SurveyInvite` | `@@index([surveyId])` | 快速查询某问卷的所有邀请码 |
| `SurveyInvite` | `@@index([code])` | 快速验证邀请码 |

## 10. 核心代码片段

### 10.1 创建邀请码（`app/api/surveys/[id]/invites/route.ts`）

```typescript
// 检查协作者数量是否已达上限
if (survey._count.collaborators >= survey.maxCollaborators) {
  return NextResponse.json(
    { error: "Collaborator limit reached" },
    { status: 400 }
  )
}

// 计算过期时间
let expiresAt: Date | null = null
if (expiresIn) {
  const days = parseInt(expiresIn)
  if (!isNaN(days)) {
    expiresAt = new Date()
    expiresAt.setDate(expiresAt.getDate() + days)
  }
}

// 生成6位邀请码（排除易混淆字符）
const code = nanoid(6)
  .toUpperCase()
  .replace(/[0O]/g, "1")
  .replace(/[IL]/g, "J")

const invite = await prisma.surveyInvite.create({
  data: {
    surveyId,
    code,
    expiresAt,
    permissions: permissions || { canEdit: false, canViewResults: false },
    createdBy: session.user.id,
  },
})
```

### 10.2 验证邀请码（`app/api/invites/validate/route.ts`）

```typescript
const invite = await prisma.surveyInvite.findFirst({
  where: { surveyId, code: code.toUpperCase() },
  include: {
    survey: { select: { id: true, title: true, userId: true } },
  },
})

if (!invite) return NextResponse.json({ error: "Invalid invite code" }, { status: 404 })
if (invite.expiresAt && new Date() > invite.expiresAt)
  return NextResponse.json({ error: "Invite expired" }, { status: 400 })
if (invite.maxUses && invite.usedCount >= invite.maxUses)
  return NextResponse.json({ error: "Invite limit reached" }, { status: 400 })

// 检查是否已是协作者或 Owner
const existing = await prisma.surveyCollaborator.findUnique({
  where: { surveyId_userId: { surveyId: invite.surveyId, userId: session.user.id } },
})
if (existing)
  return NextResponse.json({ error: "Already a collaborator" }, { status: 400 })
if (invite.survey.userId === session.user.id)
  return NextResponse.json({ error: "You are the owner" }, { status: 400 })
```

### 10.3 加入协作（`app/api/invites/join/route.ts`）

## 11. 安全与约束总结

1. **身份校验**：所有 API 均先调用 `auth()` 校验登录状态，未登录返回 `401`。
2. **Owner 唯一性**：修改/移除协作者、创建/删除邀请码等管理操作仅限 Owner，返回 `403`。
3. **邀请码防滥用**：支持过期时间与使用次数上限；加入时检查是否已达 `maxCollaborators` 上限。
4. **重复加入防护**：通过 `@@unique([surveyId, userId])` 及前置查询防止同一用户重复加入。
5. **Owner 自邀请防护**：验证接口明确拒绝 Owner 使用邀请码加入自己创建的问卷。
6. **权限降级原则**：前端与后端均遵循「只能授予不高于自身权限」的原则，防止权限提升攻击。
