# 用户认证模块

## 模块概述

用户认证模块基于 **NextAuth.js v5 (Auth.js)** 构建，提供多方式登录、JWT Session 管理、以及 Prisma 数据持久化。支持邮箱密码登录、Google OAuth 和 GitHub OAuth 三种认证方式，并为问卷系统的路由与 API 提供统一的鉴权能力。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 邮箱密码登录 | 通过 `/api/auth/login` 接口，使用 bcrypt 校验密码 |
| 邮箱密码注册 | 通过 `/api/auth/register` 接口，自动生成用户名与默认密码 |
| Google OAuth | 使用 Auth.js 内置 Google Provider |
| GitHub OAuth | 使用 Auth.js 内置 GitHub Provider |
| JWT Session | 采用 JWT 策略管理会话，token 中携带 `id` 与 `image` |
| 路由保护 | 页面级与 API 级均通过 `auth()` 校验登录状态 |

## 核心逻辑/流程

### 1. NextAuth 配置 (`lib/auth.ts`)

```ts
const { handlers, auth, signIn, signOut } = NextAuth({
  adapter: PrismaAdapter(prisma),   // Prisma 持久化
  session: { strategy: "jwt" },     // JWT 策略
  pages: { signIn: "/login" },      // 自定义登录页
  providers: [
    Google({ clientId, clientSecret }),
    GitHub({ clientId, clientSecret }),
    Credentials({
      credentials: { email: {}, password: {} },
      async authorize(credentials) {
        // 1. Zod 校验格式
        // 2. 查询 User 表（email 转小写）
        // 3. bcrypt.compare 校验密码
        // 4. 返回 { id, email, name, image }
      },
    }),
  ],
  callbacks: {
    // JWT 中注入 user.id / user.image
    async jwt({ token, user }) {
      if (user) {
        token.id = user.id
        token.image = user.image
      }
      return token
    },
    // Session 中暴露 user.id
    async session({ session, token }) {
      if (token.sub && session.user) {
        session.user.id = token.sub
        session.user.image = token.image as string | undefined
      }
      return session
    },
  },
})
```

**关键点：**
- `session.strategy: "jwt"`：不依赖数据库存储 Session，完全使用 JWT Cookie。
- `authorize` 中邮箱统一转小写，避免大小写敏感问题。
- `callbacks.jwt` / `callbacks.session` 将数据库 `user.id` 映射到前端可用的 `session.user.id`。

### 2. 注册流程 (`app/api/auth/register/route.ts`)

```ts
export async function POST(request: NextRequest) {

  // 2. 从邮箱前缀生成用户名（如 user@example.com → user）
  // 3. 若用户名冲突，自动追加 _1, _2...
  // 4. 默认密码 "Test1234"，bcrypt.hash(password, 12)
  // 5. prisma.user.create({ name, email, password })

```

**设计说明：**
- 当前注册流程极简，仅要求邮箱 + 同意条款，密码固定为 `Test1234`（演示/测试场景）。
- 用户名自动生成，无需用户填写。

## 数据结构

### Prisma Auth 模型 (`prisma/schema/auth.prisma`)

```prisma
model User {
  id            String    @id @default(cuid())
  name          String?
  email         String    @unique
  emailVerified DateTime?
  image         String?
  password      String?   // 仅邮箱密码用户有值，OAuth 用户为空
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  accounts       Account[]
  sessions       Session[]
  authenticators Authenticator[]
  surveys        Survey[]
  collaborations SurveyCollaborator[]
  surveyLogs     SurveyLog[]
}

model Account {
  id                String  @id @default(cuid())
  userId            String
  type              String
  provider          String
  providerAccountId String
  refresh_token     String? @db.Text
  access_token      String? @db.Text
  expires_at        Int?
  token_type        String?
  scope             String?
  id_token          String? @db.Text
  session_state     String?

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([provider, providerAccountId])
}

model Session {
  id           String   @id @default(cuid())
  sessionToken String   @unique
  userId       String
  expires      DateTime
  user         User     @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model VerificationToken {
  identifier String
  token      String   @unique
  expires    DateTime

  @@unique([identifier, token])
}

model Authenticator {
  credentialID         String  @unique
  userId               String
  providerAccountId    String
  credentialPublicKey  String
  counter              Int
  credentialDeviceType String
  credentialBackedUp   Boolean
  transports           String?

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, credentialID])
}
```

**说明：**
- `User.password` 可为空，OAuth 用户不存储密码。
- `Account` 表存储 OAuth Provider 返回的 token，支持多账号绑定同一用户。
- `Session` 表在 JWT 策略下**不会被 Auth.js 使用**，但保留以兼容 Adapter 接口。

## 重要实现细节

### 1. JWT Session 管理机制

- **策略选择**：`session: { strategy: "jwt" }`。
- **Token 内容**：默认包含 `sub`（用户 ID）、`email`、`name`、`picture`；通过 `jwt` callback 额外注入 `token.id` 和 `token.image`。
- **前端获取**：调用 `auth()` 或 `/api/auth/session` 均可获取到包含 `user.id` 的 Session 对象。
- **API 保护**：在 API Route 中通过 `const session = await auth()` 判断登录状态，`session?.user?.id` 即为当前用户 ID。

### 2. 路由保护机制

本项目采用**页面级权限检查**（无全局 middleware）：

- **编辑器页面** (`app/(editor)/surveys/[id]/edit/page.tsx`)：
  - 加载时并行请求 `/api/surveys/${id}` 和 `/api/auth/session`。
  - 判断当前用户是否为问卷创建者或协作者，否则显示“无权限访问”。
- **API 路由**：
  - 题目 CRUD、重排等接口均通过 `auth()` 获取 `session.user.id`，再校验问卷所有权或协作者权限。

### 3. OAuth 登录流程

1. 用户点击“Google / GitHub 登录”按钮。
2. 前端调用 NextAuth 内置 OAuth 登录（跳转到 Provider 授权页）。
3. 授权成功后，Auth.js 自动在 `Account` 表创建记录，并在 `User` 表创建/关联用户。
4. 用户被重定向回应用，JWT Cookie 自动写入浏览器。

### 4. 密码安全

- 使用 `bcryptjs` 进行哈希，cost factor 为 `12`。
- 注册时若请求体未传密码，使用默认密码 `"Test1234"`（当前为演示简化，生产环境应强制用户设置密码）。

### 5. 前端表单交互

- 使用 `react-hook-form` + `@hookform/resolvers/zod` 做表单状态管理与校验。
- 登录/注册成功后通过 `window.location.href` 硬跳转（确保 Session 状态刷新）。
- 登录页支持 `callbackUrl` 查询参数，登录成功后回到原页面。
