# 问卷编辑器模块

## 模块概述

问卷编辑器是系统的核心功能模块，提供所见即所得（WYSIWYG）的问卷设计体验。支持题型拖拽添加、题目实时编辑、拖拽排序、问卷设置管理，并集成实时协作（基于 Pusher Presence Channel）与 AI 辅助生成题目能力。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 三栏布局编辑器 | 左侧题型面板 / 中间问卷画布 / 右侧属性面板 |
| 题型拖拽添加 | 从左侧拖拽题型到画布指定位置插入 |
| 题目重排 | 拖拽已有题目调整顺序 |
| 实时协作编辑 | 多用户同时在线，题目级乐观锁定防止冲突 |
| 问卷设置 | 标题、描述、显示题目编号等 |
| 试答预览 | 支持桌面/平板/手机三种设备模拟 |
| 版本管理 | 发布历史与版本回退 |
| AI 生成题目 | 通过 AIClarifyDialog 批量生成题目 |

**布局实现要点：**
- 左栏和右栏使用 `absolute` 定位悬浮，中间画布通过 `px-[calc(20rem+2.5rem)]` 留出两侧空间。
- 整个主体区域包裹在 `DndContext` 中，支持从左侧拖拽到画布。

### 2. Zustand 状态管理设计 (`lib/editor-store.ts`)

```ts
type EditorStore = {
  survey: Survey | null       // 当前问卷完整数据
  selectedId: string | null   // 当前选中的题目 ID
  dirty: boolean              // 是否有未保存的本地变更

  // Actions
  setSurvey: (survey: Survey) => void
  selectQuestion: (id: string | null) => void
  addQuestion: (question: Question) => void
  updateQuestion: (question: Question) => void
  deleteQuestion: (id: string) => void
  reorderQuestions: (fromIndex: number, toIndex: number) => void
  updateSurveyInfo: (title: string, description: string) => void
  markSaved: () => void
}
```

**关键设计：**
- `addQuestion` 默认追加到末尾，并自动设置 `selectedId` 为新题目。
- `deleteQuestion` 删除后自动将选中态切换到下一题（或第一题）。
- `reorderQuestions` 使用 `splice` 调整数组位置，并重新计算所有题目的 `order` 字段。
- 所有变更操作均将 `dirty` 设为 `true`，保存成功后调用 `markSaved()` 重置。

### 3. 题目增删改查流程

**添加题目：**
```ts
async function handleAddQuestion(type: QuestionType) {
  const question = createQuestion(type, survey.questions.length)  // 生成默认题目
  const res = await fetch(`/api/surveys/${id}/questions`, {
    method: "POST",
    body: JSON.stringify(question),
  })
  if (res.ok) {
    const created = await res.json()
    addQuestion({ ...question, id: created.id })  // 用服务端返回的 ID 写入 store
  }
}
```

**更新题目：**
```ts
async function handleUpdateQuestion(updated: Question) {
  // 1. 检查题目是否被其他用户锁定
  const lockInfo = lockedQuestions.get(updated.id)
  if (lockInfo && lockInfo.userId !== permission.userId) {
    toast.error(`该题目正在被 ${lockInfo.userName} 编辑`)
    return
  }
  // 2. 乐观更新本地状态
  updateQuestion(updated)
  // 3. 调用 API 持久化
  const res = await fetch(`/api/surveys/${id}/questions/${updated.id}`, {
    method: "PUT",
    body: JSON.stringify({ title, description, required, config }),
  })
}
```

**删除题目：**
```ts
async function handleDeleteQuestion(qid: string) {
  const res = await fetch(`/api/surveys/${id}/questions/${qid}`, { method: "DELETE" })
  if (res.ok) deleteQuestion(qid)
}
```

### 4. 拖拽排序实现

使用 `@dnd-kit/core` + `@dnd-kit/sortable`：

```ts
// 拖拽开始：区分新题型拖拽 vs 已有题目重排
const handleDragStart = (event: DragStartEvent) => {
  if (String(active.id).startsWith("new-")) {
    setDraggingType(String(active.id).replace("new-", "") as QuestionType)
  } else {
    setDraggingId(String(active.id))
  }
}

// 拖拽经过：计算插入位置，显示指示线
const handleDragOver = (event: DragOverEvent) => {
  const overIndex = survey.questions.findIndex((q) => q.id === over.id)
  setInsertIndex(overIndex >= 0 ? overIndex : null)
}

// 拖拽结束：执行添加或重排
const handleDragEnd = async (event: DragEndEvent) => {
  if (draggingType) {
    // 新题目：在 insertIndex 位置插入
    await handleAddQuestionAtPosition(draggingType, insertIndex ?? 0)
  } else if (draggingId) {
    // 已有题目重排：本地 reorder + API 保存
    reorderQuestions(fromIndex, toIndex)
    await fetch(`/api/surveys/${id}/questions/reorder`, {
      method: "PUT",
      body: JSON.stringify({ questions: newQuestions.map((q) => ({ id: q.id, order: q.order })) }),
    })
  }
}
```

**SortableQuestionCard 组件：**
- 使用 `useSortable({ id: question.id })` 获取拖拽手柄、变换样式。
- 拖拽时透明度变为 `0.5`，释放后恢复原位或新位。
- 拖拽手柄仅在 `group-hover` 时显示，保持界面整洁。

### 5. 题目锁定机制（与实时协作的关联）

基于 **Pusher Presence Channel** 实现乐观锁定：

```ts
// 选中题目时尝试锁定
const handleSelectQuestionWithLock = (questionId: string) => {
  // 1. 立即更新本地 UI（乐观锁定）
  handleSelectQuestion(questionId)
  setOptimisticLockedId(questionId)

  // 2. 异步执行锁定（不阻塞 UI）
  const doLock = async () => {
    // 先解锁之前选中的题目
    if (selectedId && selectedId !== questionId) {
      await unlockQuestion(selectedId)
    }
    // 尝试锁定新题目
    const success = await lockQuestion(questionId)
    if (!success) {
      toast.warning("该题目正在被其他用户编辑")
    }
  }
  doLock()
}
```

**锁定状态渲染逻辑：**
```ts
const isOptimisticLocked = optimisticLockedId === q.id
const serverLockInfo = lockedQuestions.get(q.id)
const lockInfo = serverLockInfo ?? (isOptimisticLocked ? optimisticLockInfo : undefined)
const isLockedByOther = lockInfo && lockInfo.userId !== permission.userId
```

- 若题目被他人锁定：画布中该题目显示为只读（使用 `Response` 组件），并显示锁定指示器。
- 用户离开页面时，`pusher:member_removed` 事件自动释放该用户锁定的所有题目。

### 7. 自动保存策略

当前实现采用**手动保存 + 即时 API 调用**的混合策略：

- **标题/描述**：`onBlur` 时检测值是否变化，若变化则调用 `handleSaveTitleDesc` 保存。
- **题目属性**：右侧属性面板中，`onBlur` 或开关切换时即时调用 `handleUpdateQuestion`。
- **选项编辑**：`onBlur` 时对比原始值，有变化才触发 `onSave`。
- **dirty 标记**：Zustand store 中所有本地变更均标记 `dirty = true`，顶部栏可显示“未保存”状态（`editor-header.tsx` 中有此逻辑，主页面通过 `markSaved()` 在 API 成功后重置）。

> 注：当前未实现定时自动保存（debounce），所有保存均为用户操作触发的即时保存。

## 重要实现细节

### 1. 权限检查流程

页面加载时并行请求：
1. `/api/surveys/${id}` — 获取问卷基础信息。
2. `/api/auth/session` — 获取当前用户 Session。
3. 若用户非创建者，再请求 `/api/surveys/${id}/collaborators` 检查协作者身份。

### 2. 实时协作事件监听

在 `useEffect` 中注册 5 类 Pusher 事件：
- `QUESTION_UPDATED` — 远程题目更新
- `QUESTION_CREATED` — 远程新增题目
- `QUESTION_DELETED` — 远程删除题目
- `QUESTIONS_REORDERED` — 远程重排
- `SURVEY_UPDATED` — 远程问卷信息更新

所有事件均通过 `fromUserId === permission.userId` 过滤掉自己触发的变更。

### 3. 空画布拖拽区

当问卷无题目时，画布中央显示 `empty-canvas-dropzone`，拖拽新题型到此处会在位置 `0` 插入题目。

### 4. 题目编辑器组件 (`QuestionEditor`)

```ts
function QuestionEditor({ question, onUpdate, onSave, onDelete }) {

  return (
    <div>

```

`hasChangesRef` 用于追踪是否有未保存的变更，避免无意义的 API 调用。

### 5. 移动端适配

- 矩阵题 (`MATRIX_SINGLE`) 在移动端使用卡片式纵向布局替代表格。
- 图片选择题支持自定义 `aspectRatio`（3:4、4:3、1:1、16:9、9:16），适配不同屏幕。
