# 实时协作模块

## 模块概述

实时协作模块基于 **Pusher Presence Channel** 实现多用户同时编辑同一份问卷时的实时同步与冲突避免。核心能力包括：在线成员感知、题目乐观锁定、编辑事件广播、异常退出自动恢复。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| **在线成员列表** | 实时展示当前正在编辑问卷的所有用户 |
| **题目乐观锁定** | 用户选中题目即锁定，阻止他人同时编辑 |
| **实时内容同步** | 题目增删改、重排、问卷信息更新即时广播 |
| **异常恢复** | 用户断线或关闭页面时，自动释放其锁定的题目 |
| **权限控制** | 仅问卷所有者或具备编辑权限的协作者可参与协作 |

## 核心逻辑/流程

### 1. 协作架构（Pusher Presence Channel）

```
┌─────────────┐         ┌─────────────┐
│   用户 A     │◄───────►│   用户 B     │
│  (编辑者)    │         │  (编辑者)    │
└──────┬──────┘         └──────┬──────┘
       │                       │
       │   WebSocket (Pusher)  │
       └───────────┬───────────┘
                   │
           ┌───────▼───────┐
           │  Presence      │
           │  Channel       │
           │  presence-survey-{id}
           └───────┬───────┘
                   │
           ┌───────▼───────┐
           │  Pusher 服务端  │
           │  维护成员列表   │
           └───────────────┘
```

Presence Channel 相比普通 Channel 的优势：
- 自动维护在线成员列表（`subscription_succeeded`、`member_added`、`member_removed`）
- 每个成员可携带自定义信息（`user_info`，如 name、image）
- 需要服务端鉴权，天然具备权限控制能力

### 2. 在线成员管理

**加入流程：**

```
用户打开问卷编辑页
    │
    ▼
useSurveyCollaboration(surveyId, userId) 初始化
    │
    ▼
Pusher 订阅 presence-survey-{surveyId}
    │
    ▼
POST /api/pusher/auth  ← 鉴权：验证用户是否为所有者/协作者
    │
    ▼
pusher:subscription_succeeded  ← 获取完整成员列表，渲染在线头像
```

**离开流程（关键）：**

```
用户关闭页面 / 断线
    │
    ▼
Pusher 检测到连接断开
    │
    ▼
pusher:member_removed 事件广播给所有在线成员
    │
    ▼
前端自动从成员列表移除该用户
    │
    ▼
前端本地清理该用户锁定的题目
    │
    ▼
调用 POST /api/surveys/collaboration/unlock-all
    │         { surveyId, userId: 离开的用户ID }
    ▼
服务端批量解锁该用户所有题目，并广播 unlock-all 事件
```

**关键代码：成员状态管理**

```ts
// hooks/use-survey-collaboration.ts
channel.bind("pusher:subscription_succeeded", (data) => {
  setIsConnected(true)
  const membersMap = new Map()
  Object.entries(data.members).forEach(([userId, member]) => {
    membersMap.set(userId, { userId, name: member.name, image: member.image })
  })
  setMembers(membersMap)
})

channel.bind("pusher:member_added", (member) => {
  setMembers((prev) => {
    const next = new Map(prev)
    next.set(member.id, { userId: member.id, name: member.info?.name })
    return next
  })
})

channel.bind("pusher:member_removed", (member) => {
  // 1. 移除成员
  setMembers((prev) => { /* ... */ })
  // 2. 本地清理锁定
  setLockedQuestions((prev) => {
    const next = new Map(prev)
    for (const [qid, lock] of next.entries()) {
      if (lock.userId === member.id) next.delete(qid)
    }
    return next
  })
  // 3. 服务端批量解锁
  fetch("/api/surveys/collaboration/unlock-all", {
    method: "POST",
    body: JSON.stringify({ surveyId, userId: member.id }),
  })
})
```

### 3. 题目乐观锁定机制

**锁定规则：**
- 用户**选中**某道题目时，前端调用 `lockQuestion(questionId)`
- 若题目已被他人锁定，API 返回 `409 Conflict`，前端提示「xxx 正在编辑」
- 用户**切换到其他题目或离开页面**时，调用 `unlockQuestion(questionId)` 释放
- 用户异常退出（断线/关闭标签页）时，由 `member_removed` 触发批量解锁

**锁定状态流转：**

```
未锁定 ──► 选中题目 ──► 调用 lock API ──► 数据库写入 lockedBy + lockedAt
                                              │
                                              ▼
                                         广播 question-locked 事件
                                              │
                                              ▼
                                         所有客户端更新 lockedQuestions Map
                                              │
                    ┌──────────────────────────┘
                    │
              编辑完成/切换题目
                    │
                    ▼
            调用 unlock API ──► 数据库清空 lockedBy
                    │
                    ▼
            广播 question-unlocked 事件
```

**关键代码：锁定 API**

```ts
// app/api/surveys/collaboration/lock/route.ts
export async function POST(request: Request) {
  // 1. 权限校验（所有者或 canEdit 协作者）
  // 2. 检查题目是否已被他人锁定
  if (question.lockedBy && question.lockedBy !== session.user.id) {
    return NextResponse.json(
      { error: "题目已被锁定", lockedBy: lockedByUser?.name },
      { status: 409 }
    )
  }
  // 3. 写入锁定信息
  await prisma.question.update({
    where: { id: questionId },
    data: { lockedBy: session.user.id, lockedAt: new Date() },
  })
  // 4. 广播锁定事件
  await pusherServer.trigger(channel, COLLABORATION_EVENTS.QUESTION_LOCKED, {
    questionId, userId: session.user.id, userName: user?.name, lockedAt: new Date().toISOString(),
  })
}
```

### 4. 实时同步事件

`lib/pusher.ts` 定义了完整的事件常量：

```ts
export const COLLABORATION_EVENTS = {
  // 成员管理
  MEMBER_JOINED: "member-joined",
  MEMBER_LEFT: "member-left",

  // 题目锁定
  QUESTION_LOCKED: "question-locked",
  QUESTION_UNLOCKED: "question-unlocked",
  QUESTIONS_UNLOCK_ALL: "questions-unlock-all",

  // 题目内容变更
  QUESTION_UPDATED: "question-updated",
  QUESTION_CREATED: "question-created",
  QUESTION_DELETED: "question-deleted",
  QUESTIONS_REORDERED: "questions-reordered",

  // 问卷级变更
  SURVEY_UPDATED: "survey-updated",
  SURVEY_PUBLISHED: "survey-published",
} as const
```

内容同步由编辑器组件通过 `onEvent` 注册监听器实现：

```ts
const { onEvent } = useSurveyCollaboration(surveyId, userId)

useEffect(() => {
  const unbind = onEvent(COLLABORATION_EVENTS.QUESTION_UPDATED, (data) => {
    // 更新本地题目状态（排除自己触发的事件）
    if (data.fromUserId !== currentUserId) {
      updateQuestion(data.question)
    }
  })
  return unbind
}, [onEvent])
```

### 5. 冲突处理策略

| 场景 | 策略 |
|------|------|
| 同时选中同一题 | 先到达的锁定请求成功，后到达的收到 `409`，前端提示 |
| 用户 A 编辑时用户 B 也编辑 | 乐观锁定阻止 B 进入编辑态，B 只能查看 |
| 用户断线未释放锁 | `member_removed` 触发自动批量解锁（本地+服务端） |
| 用户恶意不释放锁 | 所有者可通过「强制解锁」或刷新页面恢复（暂未实现显式强制解锁 UI） |

  image: string | null
  joinedAt: string   // ISO 时间戳

### 协作 Hook 返回状态

```ts
interface CollaborationState {
  members: Map<string, MemberInfo>         // 在线成员列表
  lockedQuestions: Map<string, LockInfo>   // 当前被锁定的题目
  isConnected: boolean                     // Pusher 连接状态

interface CollaborationActions {
  lockQuestion: (questionId: string) => Promise<boolean>

}
```

## 重要实现细节

### 1. Pusher 鉴权流程

Presence Channel 必须通过服务端鉴权，防止未授权用户监听频道：

```ts
// app/api/pusher/auth/route.ts
export async function POST(request: Request) {
  const session = await auth()
  if (!session?.user?.id) return 401

  const { socket_id, channel_name } = await request.formData()
  const surveyId = channel_name.match(/^presence-survey-(.+)$/)[1]

  // 验证用户是否有权限访问该问卷
  const survey = await prisma.survey.findUnique({
    where: { id: surveyId },
    include: { collaborators: { where: { userId: session.user.id } } },
  })

  const isOwner = survey.userId === session.user.id
  const isCollaborator = survey.collaborators.length > 0
  if (!isOwner && !isCollaborator) return 403

  // 生成 Presence Channel 认证签名
  const authResponse = pusherServer.authorizeChannel(socket_id, channel_name, {
    user_id: session.user.id,
    user_info: { name: session.user.name, image: session.user.image },
  })

  return NextResponse.json(authResponse)
}
```

### 2. 协作 Hook 的使用方式

```tsx
// 在问卷编辑器页面中使用
function SurveyEditor({ surveyId, userId }) {
  const {
    members,
    lockedQuestions,
    isConnected,
    currentUser,
    lockQuestion,
    unlockQuestion,
    onEvent,
  } = useSurveyCollaboration(surveyId, userId)

  // 选中题目时锁定
  const handleSelectQuestion = async (questionId: string) => {
    const success = await lockQuestion(questionId)
    if (!success) {
      toast.error("该题目正在被其他人编辑")
      return
    }
    setSelectedQuestionId(questionId)
  }

  // 渲染锁定指示器
  return (
    <div>
      <OnlineMembers members={members} currentUserId={currentUser?.userId} />
      {questions.map((q) => (
        <QuestionCard
          key={q.id}
          lockInfo={lockedQuestions.get(q.id)}
          isLockedByMe={lockedQuestions.get(q.id)?.userId === currentUser?.userId}
        />
      ))}
    </div>
  )
}
```

### 3. 数据库锁定字段

`Question` 模型包含两个协作专用字段：

```prisma
model Question {
  // ...
  lockedBy String?   // 正在编辑的用户ID
  lockedAt DateTime? // 锁定时间（可用于判断锁是否超时）
}
```

当前实现未设置锁超时机制，依赖 Pusher 的 `member_removed` 事件做自动清理。若 Pusher 事件丢失（极端情况），锁可能残留，可通过刷新页面或所有者调用 `unlock-all` 恢复。

### 4. 单例模式管理 Pusher 客户端

为避免多次实例化，客户端 Pusher 使用模块级单例：

```ts
// hooks/use-survey-collaboration.ts
let pusherClient: Pusher | null = null

function getPusherClient(): Pusher | null {
  if (typeof window === "undefined") return null // SSR 安全
  if (!pusherClient) {
    pusherClient = new Pusher(process.env.NEXT_PUBLIC_PUSHER_KEY!, {
      cluster: process.env.NEXT_PUBLIC_PUSHER_CLUSTER!,
      authEndpoint: "/api/pusher/auth",
    })
  }
  return pusherClient
}
```
